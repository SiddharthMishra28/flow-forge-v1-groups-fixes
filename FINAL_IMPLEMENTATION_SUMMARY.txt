CLARIFICATIONS
================================================================================

1. SERVER STARTUP BEHAVIOR:
   ------------------------
   Q: Does restart resume all 50 queued flows at once?
   A: NO! The system intelligently processes based on available capacity:
      - On startup, checks ThreadPoolTaskExecutor available slots
      - Only processes up to available capacity (e.g., 5 slots = 5 flows)
      - Remaining flows stay queued
      - Scheduled polling (every 30s) picks them up as capacity frees

2. RETRY MECHANISM:
   ----------------
   Q: Does it retry resuming queued flows 3 times?
   A: YES, but only for QUEUE PROCESSING failures:
      - Retries if thread pool submission fails
      - Retries if database errors occur during queue processing
      - NOT for flow execution failures (handled by FlowExecutionService)
      - After 3 failed attempts, flow is removed from queue
