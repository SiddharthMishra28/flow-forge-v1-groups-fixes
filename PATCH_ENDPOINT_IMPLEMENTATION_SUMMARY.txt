OVERVIEW
--------
Successfully implemented a new PATCH endpoint that allows modification of 
FlowExecutionIds within an Artifact. The endpoint supports both adding and 
removing FlowExecution run IDs in a single operation.

API ENDPOINT
------------
PATCH /api/artifacts/{id}

Request Body (ArtifactPatchDto):
{
  "addRunIds": ["uuid1", "uuid2", ...],      // Optional: List of UUIDs to add
  "removeRunIds": ["uuid3", "uuid4", ...]    // Optional: List of UUIDs to remove
}

Response: ArtifactDto (200 OK)
{
  "id": 1,
  "artifactName": "Test Artifact",
  "runIds": ["uuid1", "uuid2", ...],
  "createdAt": "2026-03-12T18:00:00",
  "updatedAt": "2026-03-12T18:15:00"
}

Error Responses:
- 400 Bad Request: Invalid run IDs to add (non-existent FlowExecutions)
- 404 Not Found: Artifact with specified ID not found

FEATURES IMPLEMENTED
--------------------
1. Add FlowExecution IDs to an existing Artifact
   - Validates that all IDs to add exist in the FlowExecution repository
   - Prevents duplicate entries automatically
   
2. Remove FlowExecution IDs from an existing Artifact
   - Safely removes specified IDs from the artifact's runIds list
   
3. Combined Add and Remove operations in a single request
   - Process order: First adds new IDs, then removes specified IDs
   
4. Validation & Error Handling
   - Artifact existence check
   - FlowExecution existence validation for IDs being added
   - Proper exception handling with meaningful error messages

VALIDATION LOGIC
----------------
1. Artifact Validation:
   - Checks if artifact exists before processing
   - Throws IllegalArgumentException if not found

2. FlowExecution Validation (for additions):
   - Validates all run IDs to add exist in FlowExecution table
   - Throws IllegalArgumentException with list of missing IDs if validation fails

3. Duplicate Prevention:
   - Automatically skips adding run IDs that already exist in the artifact
   - Maintains data integrity
